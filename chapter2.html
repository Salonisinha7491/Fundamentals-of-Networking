OSI(Open Source Interconnection)

Need of layered architecture in OSI.

  1. Modularity
  2. Flexibility
  3. Independent Nature of layer
  4. To reduce the complexity in data communication
  5. To provide standarization
  6. Reliability Approach
  7. Load Distribution
  8. Tasks are categorise into different sub-task from source to destination
  9. Specific implementation of its functions can be modified or replace
  10. For the abstract representation of data communication process

layer in OSI model

layer nodes links
7 Application layer Application layer
6 Presentation layer Presentation layer
5 Session layer Session layer
4 Transport layer Transport layer
3 Network layer Network layer
2 Data link layer Data link layer
1 Physical layer Physical layer

Application layer

  • Message preparation or message composition.
  • File Transfer,access,management.
  • Mail Services.
  • Allow access to network resource.

Presentation layer

  • Data Encryption or Decryption.
  • Transport services.
  • Compression Services.

Session layer

  • Dialog Control.
  • Synchronization.
  • Session management.

Transport layer

  • Service Point Delivery.
    • End-to-End delivery.
    • Process-to-Process delivery.
  • Segmentation and Reassemble.
  • Connection Control.
    • Connection- oriented (TCP)
    • Connectionless- oriented (UDP)
  • Flow Control.
  • Error Control

Network layer

  • Routing.
  • Packetization.
  • Congestion control.
  • Logical address.
  • Accounting and Billing.
  • Address transform technique.
    • Logical to physical (ARP)
    • Physical to logical (RARP)

Data link layer

  • Framing
  • Physical Addressing
  • Access Control
    • Single station model [Point to Point]
    • Multiple access model [Multipoint]
  • Node to Node delivery
  • Flow control
  • Error contol

Physical layer

  • Representation of bits
  • Signalling
  • Physical characteristic or transmission media interface
  • Transmission mode
    • Simplex
    • Half duplex
    • Full duplex
  • Transmission rate
  • Bit synchronization
  • Line configuration
    • Point to point
    • Multipoint
  • Physical Topology
  • Multiplexing
  • Transmission type
    • Serial
    • Parallel
